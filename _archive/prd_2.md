# Crazy Match - 世界杯比赛预测平台

> PRD v0.1 · 2026-05-29

---

## 1. 项目概述

**项目名称**: Crazy Match
**项目类型**: Web 应用（预测型工具）
**核心定位**: 基于历史数据和球员实力，对世界杯比赛结果进行预测分析
**目标用户**: 球迷、博彩爱好者、数据分析爱好者

---

## 2. 核心功能

### 2.1 比赛数据获取

- **2026年世界杯完整赛程**：包含所有小组赛、淘汰赛（含具体日期、时间、场地、对阵双方）
- **数据来源**: 实时爬取 FIFA 官方数据或第三方世界杯数据 API
- **数据更新策略**: 开赛前定时同步，比赛期间实时更新

### 2.2 历史成绩查询

- **各队过往世界杯成绩**：涵盖 1930 年至今所有参赛队伍的战绩
- **关键指标**:
  - 参赛届数
  - 总胜/平/负场次
  - 总进球/失球
  - 最佳成绩（冠军/亚军/四强等）
  - 近年表现（最近 3 届战绩）

### 2.3 预测模型

#### 模型一：历史战绩预测

基于各队过往世界杯整体表现计算基础胜率：

```
战绩分 = (胜场数 × 3 + 平场数 × 1) / 总场次 × 权重系数
近期加成 = 最近一届成绩 × 调整系数
预测胜率 = (战绩分 + 近期加成) / 对手战绩分
```

#### 模型二：球员实力评估（100分制）

综合考量各队球员的：

- **个人能力值**：参考 FIFA 球员评分（0-100）
- **阵容完整性**：主力球员健康状态、年龄结构
- **位置分布**：关键位置（前锋、中场、门将）人员储备
- **大赛经验**：球员过往世界杯/大赛出场次数

计算公式：

```
球队实力分 = Σ(球员评分 × 位置权重) × 大赛经验系数 × 阵容完整度
```

### 2.4 数据展示

- 球队信息卡片：队徽、排名、关键数据
- 比赛预测面板：双方实力对比、胜率预测
- 历史交锋记录

### 3.4 核心交互

- 日历控件切换日期 → 更新比赛列表
- 点击比赛卡片 → 展开预测详情面板
- Tab 切换小组赛/淘汰赛阶段
- 切换预测模型（历史模型 / 球员模型）

### 3.5 社交功能（无数据库、无登录）

**设计原则**: 所有社交功能基于 localStorage + 分享链接实现，无需后端数据库和用户登录系统。

#### 3.5.1 匿名预测 + localStorage 存储

- 用户提交预测时只需输入昵称（不注册）
- 预测数据存储在 localStorage
- 每场比赛开奖后自动计算准确率
- 用户可查看"我的预测"记录

#### 3.5.2 匿名排行榜

- 纯前端统计：从 localStorage 读取所有预测记录
- 维度：总准确率 / 今日准确率 / 连胜次数
- 展示前 20 名，匿名展示昵称
- 刷新页面后基于本地数据重新计算

#### 3.5.3 预测分享链接

- 生成的 URL 包含预测结果参数
- 示例：`crazymatch.app/p/bra-cro?winner=bra`
- 接收者打开链接可查看预测内容
- 投票/评论交由社交平台（微信/微博）处理

#### 3.5.4 晒单图片生成

- Canvas 渲染生成可分享的图片
- 包含：比赛信息、预测结果、昵称、生成时间
- 适配微信/微博分享尺寸（1:1 或 9:16）
- 内置引导语"扫码来 Crazy Match 预测"

**布局**:

### 4.3 API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/matches` | GET | 获取比赛列表/赛程 |
| `/api/teams` | GET | 获取球队列表 |
| `/api/teams/:id` | GET | 获取球队详情+历史成绩 |
| `/api/predict` | POST | 提交预测请求 |
| `/api/predict/:matchId` | GET | 获取单场比赛预测结果 |

### 4.4 数据模型

```typescript
// 球队
interface Team {
  id: string;
  name: string;
  code: string;       // 如 "BRA", "ARG"
  logo: string;        // 队徽 URL
  historicRecord: HistoricRecord;
  currentSquad: Player[];
  strengthScore: number; // 100分制
}

// 球员
interface Player {
  id: string;
  name: string;
  position: 'GK' | 'DEF' | 'MID' | 'FWD';
  rating: number;      // 0-100
  matches: number;     // 大赛出场次数
}

// 比赛
interface Match {
  id: string;
  date: string;
  stage: 'group' | 'round16' | 'quarter' | 'semi' | 'final';
  teamA: Team;
  teamB: Team;
  prediction?: Prediction;
}

// 预测结果
interface Prediction {
  model: 'historic' | 'squad';
  winProbability: { teamA: number; teamB: number; draw: number };
  strengthDiff: number;
  factors: string[];    // 预测依据
}

// 用户预测（localStorage 存储）
interface UserPrediction {
  id: string;
  matchId: string;
  nickname: string;
  predictedWinner: 'teamA' | 'teamB' | 'draw';
  predictedScore?: { teamA: number; teamB: number };
  createdAt: string;
}

// 用户积分（localStorage 存储）
interface UserStats {
  nickname: string;
  totalPredictions: number;
  correctPredictions: number;
  currentStreak: number;
  longestStreak: number;
}
```

## 5. 数据获取方案

### 5.1 数据来源

| 数据类型 | 来源 | 状态 | 说明 |
|----------|------|------|------|
| 2026世界杯赛程 | **模拟数据** | ✅ 已确认 | MVP阶段使用预设赛程，真实赛程待抽签后接入 |
| 小组赛对阵 | **模拟数据** | ✅ 已确认 | 32强按分档预设，A-H组对阵模拟 |
| 历史战绩 | **模拟数据** | ✅ 已确认 | 主要参赛国历史数据（1930-2022） |
| 淘汰赛对阵 | **模拟数据** | ✅ 已确认 | 16强起按预设逻辑推演 |
| 球员评分 | **模拟数据** | ✅ 已确认 | MVP按位置+星级模拟真实球员评分 |
| 真实赛程 | FIFA官方 / 维基百科 | ⏳ 待接入 | 抽签完成后（预计2024年12月） |
| 真实历史数据 | Wikipedia / OpenData | ⏳ 待接入 | 需结构化清洗，工作量大 |
| 真实球员评分 | Transfermarkt / SOFIFA | ⏳ 待评估 | 付费API，评估成本后决定 |
| 真实球队阵容 | 各大体育媒体 | ⏳ 待接入 | 开赛前1个月公布后接入 |

### 5.2 数据维护策略

| 阶段 | 时间 | 数据策略 |
|------|------|----------|
| MVP开发期 | 现在 - 开赛前 | 纯模拟数据，保证功能完整 |
| 抽签后 | ~2024年12月 | 接入真实赛程+分档，保留模拟阵容 |
| 开幕前3个月 | ~2026年3月 | 接入真实球队阵容+球员评分 |
| 比赛期间 | 2026年6月-7月 | 每日更新比分，实时计算预测准确率 |

### 5.4 数据质量标准

- 模拟数据必须覆盖 **32 强全部参赛队**
- 每队至少包含 **23 名模拟球员**（符合真实阵容人数）
- 模拟数据格式与真实 API 返回格式 **完全一致**，便于后续替换

## 7. 约束与限制

- 本项目仅支持 **2026 世界杯**（美国/加拿大/墨西哥联合举办）
- MVP 阶段球员数据使用 **模拟数据**，Phase 2 接入真实评分
- 预测结果仅供娱乐参考，不构成任何投注建议
- **社交功能完全无数据库、无登录**，依赖 localStorage + 外部社交平台
